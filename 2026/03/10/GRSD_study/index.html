1.1 ymal文档当中的参数

1.1.1 项目的整体 pipeline 是什么？

针对清洗之后的数据，item text → LLM encoder(T5) → item embedding(2048，离散) → Residual quantization(量化token) → Semantic ID(离散序列向量数据) → Transformer (TIGER) → 生成推荐

1.1.2 为什么用 LLM 生成 embedding？

item数据有语义文本，LLM又能够捕捉到语义信息，同时embedding能够展示物品的相似度

1.1.3 为什么使用 T5Encoder？

开源，能够处理语义信息，而且我们仅仅需要encoder不需要decoder

1.1.4 embedding 是怎么得到的？

对于T5的token embedding输出做了平均的池化
