nvitop参数

SM

SM 利用率（Streaming Multiprocessor，CUDA 核心在干活的忙碌程度），训练时这个高，通常说明 GPU 计算跑满了

GMBW

显存带宽利用率（Global Memory Bandwidth），这个高说明“搬数据/访存”很重，可能受带宽瓶颈限制

CPU

这个进程占用的 CPU 百分比，如果 DataLoader/解码/增强重，这里会高

MEM

系统内存占用比例（不是显存，是主机 RAM）

Utl

GPU-Util（有时被简写成 UTL / UTIL）：GPU 总体利用率（0~100%）；%SM，更细化的“计算单元利用率”；%GMBW，更细化的“显存带宽利用率”

Adam和Adagrad优化器

Adam优化器，包含一阶以及二阶的动量矩，本质上就是梯度的平均和平方的平均，表示如下：

mt=β1mt−1+(1−β1)gt\begin{gathered} m_{t} = \beta_{1}m_{t-1} + (1-\beta_{1})g_{t} \end{gathered} mt​=β1​mt−1​+(1−β1​)gt​​

vt=β2vt−1+(1−β2)gt2\begin{gathered} v_{t} = \beta_{2}v_{t-1} + (1-\beta_{2})g_{t}^{2} \end{gathered} vt​=β2​vt−1​+(1−β2​)gt2​​

θ⟵θ=α∗mtvt+ξ\begin{gathered} \theta \longleftarrow \theta = \frac{\alpha *m_{t}}{\sqrt{v_{t}+\xi } } \end{gathered} θ⟵θ=vt​+ξ​α∗mt​​​

故而Adam可以针对大梯度使用较小的学习率，反之亦然；但是针对Adamgrad不同的点在于他对于稀疏的更新的更加的好，它会将稀疏不常用的token使用较大的lr，越长用此时的lr就会越小，这就适合稀疏的更新。
其次我们设置的显性的固定的rl是缩放系数罢了，真正的步长由其共同决定。
